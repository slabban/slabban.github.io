• Object Tracking With YOLO

This project was an deep dive into the world of Multi-Object Tracking (MOT) where I employed the powerful Extended Kalman Filter (EKF) and an Intersection Over Union (IoU) association approach to track objects in a video stream. I was able to deploy the algorithm in the Robot Operating System (ROS) Middleware, which allowed me to tap into the vast ecosystem of ROS packages and tools.

My inspiration for this project came from this article by Jeremy Cohen. I came across it over a year ago and thought: "I know how to design a Kalman Filter and have worked on Multiple Object Tracking, I should give this a shot." So, I did!

I experimented with both YOLOv3 and YOLOv4, comparing results as seen in the video above. YOLOv4 was better at association, but YOLOv3 performed detections faster, which allowed the Kalman Filter to produce smoother estimates. Using YOLOv3, I achieved a detection output rate of 17 HZ and estimate rate of 50 HZ on my dingy but reliable 2017 Laptop with a 1060 GPU, while YOLOv4 brought the rate down to 7.5 HZ but produced more accurate results. I am curious about how better-calibrated input images could further improve performance.

My algorithm, like most MOTs, follows these 5 steps:

  • Detection
  • Association
  • Track Identity Creation
  • Estimation
  • Track Identity Destruction

There are some drawbacks to classical approaches like the Extended Kalman Filter sucha as tracking loss through ID swaps via occlusion. SORT, uses the Hungarian Algorithm for association, and can reduce these issues, DeepSort offers a more robust solution to the occlusion issue by leveraging feature maps from a Recurent Neural Network (RNN) for association. Learn more about DeepSort here.

I had a great time with this project and learned so much from the open-source community. I hope this project can contribute in return and inspire others to take on similar projects!
